# 前端开发计划

基于《系统开发说明书》与《VibeCoding原则》，制定以下前端开发计划。整个开发过程切分为独立的模块化步骤，每进行一个步骤都需要完成后自测闭环，并更新进度文档。

## 第一阶段：完成并重构项目基础环境与公共组件（已部分完成）
- **目标**：梳理目前代码（已有的demo阶段），建立完整、规范的项目基础架构。
- **内容**：
  1. 确认路由结构：`app/(auth)`、`app/(dashboard)` 的页面布局，以及 `app/audit`（提取页面详情）。
  2. 全局状态管理：选型和集成基础配置（如zustand或context，配合Supabase及API客户端）。
  3. API 请求封装：基于 axios 或 fetch 封装统一的响应与异常处理机制。
  4. 设计规范与基础 UI 组件：引入或完善基于 TailwindCSS/Shadcn 等方案的基础UI，保证样式风格统一与高可用。

## 第二阶段：用户认证与权限管理模块
- **目标**：实现用户注册与登录功能，确保页面与数据隔离。
- **内容**：
  1. 注册功能页面及表单校验。
  2. 登录功能页面及会话管理。
  3. 路由守卫：普通用户的鉴权机制。

## 第三阶段：任务与文件管理模块 (Dashboard)
- **目标**：实现文件上传及历史任务管理功能。
- **内容**：
  1. 任务列表页面。
  2. 文件上传组件 (支持多文件，文件夹拖拽)。
  3. 批量流转操作。

## 第四阶段：提示词组与模型配置模块
- **目标**：实现对LLM的提示词组和参数配置。
- **内容**：
  1. 模型配置 CRUD与连通性测试。
  2. 提示词组 CRUD 与 Excel导入导出。
  3. 单字段调试区域。

## 第五阶段：核心提取详情页 (Task Details)
- **目标**：完成三栏式的详情交互页面。
- **内容**：
  1. 左侧：Task文件树及状态管理。
  2. 中栏：文件多格式预览及高亮定位检索。
  3. 右侧：各类字段结果展示及出处关联交互。

## 第六阶段：最终联调测试与功能验收
- **目标**：前后端接口对接及完善异常处理。
